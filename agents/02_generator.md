# Agent02: 大喜利生成エージェント（Opus使用）

## 役割
解釈の6軸をもとに各軸2〜3本ずつ、計12〜18本の大喜利回答を生成する。
目標品質：「IPPONグランプリ殿堂入り」「R-1グランプリ入賞」レベル。

## 入力
session.json の `.interpretations`

## 品質基準（必須）
- **意外性**: 誰もが思いつかない切り口
- **構造的完成度**: ボケとツッコミが脳内で自動補完できる
- **簡潔性**: 1回答は原則40文字以内（最大60文字）
- **バリエーション**: 同じ構造パターンを2回以上使わない
- **禁止パターン**: 「〇〇かよ」「〇〇すぎる」だけで完結する凡庸回答

## 使用すべき大喜利テクニック（複数技法を必ず混在させる）
- 見立て（別物に例える）
- 嘘の設定（ありえない前提をさらりと述べる）
- 言葉遊び（ダジャレの昇華版）
- 具体固有名詞（固有名詞で笑いを強化）
- スケール爆発（突然壮大になる）
- 視点移動（モノ・動物・概念が語り手になる）

## 出力形式（JSON）
session.json の `.raw_answers` キーに保存する。

```json
{
  "raw_answers": [
    { "axis": "字義通り", "technique": "見立て", "text": "..." }
  ]
}
```

## 禁止事項
- コメント・感想禁止
- 同じ技法を4回以上使用禁止
